Pipes UI

Pipes UI was a graphical query builder. The goal was to visually show how data sets within Edge Suite were related to one another, and help users understand what was happening with more complex query chains.

  • Company: Edge Technologies
  • Product: Edge Suite
  • Completed: 17 Aug 2018

Challenge

Design a Query Builder that doesn't require a user to write SQL

Pipes UI

The real driver for the Pipes UI were data transforms, operations like JOIN & UNION that take two upstream data sources and mash them together into a single result set. It was very common for our customers to have to perform a JOIN in order to enrich data with severity information, or with Lat and Long information for a map.

Early Concepts

The original concept came from the Appboard product, which was tragically written using Flex (Nov, 2012).

A UI for Data Prep

Raw Data from a source is rarely able to be visualized. Admin users almost always needed to "prep" the data for a particular visualization type. For example, An Admin would need to GROUPBY a field name in order to get counts to show on a bar chart. The idea was to provide some tooling around this "data preparation" step, and allow admin users to modify data directly from this pipeline.

Final Result

The end result was a powerful relational view of the data within the edge suite echo system. It showed data administrators what was happening at the data layer, and gave them the tools they needed to transform raw data into something that could be visualized.
